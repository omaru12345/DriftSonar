# DriftSonar — 収益化方針メモ

**ステータス**: 検討中（App Store 掲載を優先、収益化は後回し）
**関連 ISSUE**: EP-022（収益化モデル構築）
**関連コンセプト**: `docs/concept.md`（差別化＝個々人の秘密の話に閉じた SNS）

## 基本スタンス

1. **App Store 掲載を最優先**。収益化はリリース後にユーザー反応を見てから決める。
2. 収益化は IAP・設定変更で**後付けできる**ため、リリース前に作り込まない。
3. 収益モデルは**コンセプト（サーバーレス・無トラッキング・edge 完結）と整合する手段を優先**する。

## 選択肢の比較

| 手段 | privacy 整合 | サーバー要否 | コンセプト適合 | 備考 |
|------|:-----------:|:-----------:|:-------------:|------|
| **買い切り有料アプリ** | ◎ | 不要 | ◎ | 最もシンプル・思想と完全整合 |
| **無料 + プレミアム IAP** | ◎ | 不要 | ◎ | 端末内完結の機能を有料化 |
| **投げ銭 / Tip（Consumable IAP）** | ◎ | 不要 | ◎ | OSS 的プロダクトと好相性 |
| **広告（AdMob 等）** | ✗ | 要（通信） | ✗ | 下記の理由で最終手段 |

## 広告モデルの問題点（最終手段とする理由）

広告を入れた瞬間に、今まで築いた差別化を**自ら壊す**ことになる。

- **第三者 SDK 追加**: 現状「サードパーティ SDK ゼロ」が崩れる。
- **トラッキング発生**: 広告 ID 収集 → ATT 許可ダイアログ必須、`NSPrivacyTracking` を `true` に変更。
- **プライバシー資産の全面改訂**: `DriftSonarApp/.../PrivacyInfo.xcprivacy`（収集データなし）と `docs/privacy-policy.md`（「広告 SDK・トラッキング一切なし」）を書き直す必要がある。
- **オフライン思想と矛盾**: 広告取得に通信が必要 = 「WiFi なし・オフグリッド・edge 完結」というコアバリューに反する。
- **ユーザー層との相性**: 想定ユーザー（プライバシー志向・反監視・オフグリッド層）はそもそも広告を嫌う。

→ 広告は「privacy 整合を捨てる覚悟」がある場合のみ。原則は非採用。

## 推奨ロードマップ

```
1. 無料・無広告で App Store へ（EP-019: 実機検証 → 配布設定）
2. リリース後、ユーザー反応を見る
3. 収益化方針を決定（TASK-117）
   第一候補 = 買い切り / プレミアム IAP / 投げ銭 のいずれか
4. StoreKit 2 で端末内完結の購入フローを技術検証（TASK-119）
```

## プレミアム機能候補（IAP を採る場合のたたき台）

> サーバーレス原則を崩さない＝**端末内で完結する機能**であることが条件。詳細な洗い出しは TASK-118 で行う。

- 見た目系: カスタムテーマ、追加アプリアイコン、配色プリセット
- 機能系: 伝播統計の可視化、メッセージ/キャッシュ保存上限の解放、匿名投稿の拡張オプション
